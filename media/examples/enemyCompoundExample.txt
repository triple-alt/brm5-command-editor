    //TRIGGERS(8)
        //compoundPassiveProximity
            //pre: compoundClock must be inactive. Player-whitelisted. Must be reset by a stopMethod so players can trigger it.
            //post: Area is now loaded, compoundClock is active.
            //description: when activated, do something, start the clock signal, then block the boolean false trigger.

            //When player enters this trigger zone, spawn initial bots.
            trigger activate 650 compoundInitialSpawn
            //Start the clock signal.
            trigger whitelist 650 compoundClock IsLooping true
            //Only after starting the clock signal, block the boolean's "false" trigger.
            //Ensure it is blocked before bot signals are sent.
            move 650 %compoundBooleanPart 80 1018 -6 -0 0 0
        //compoundClock
            //pre: must be activated by an IsLooping true executable by compoundPassiveProximity.
            //post: on completion, loop.
            //description: A clock signal with a 2 second frequency. First interval is overridden by compoundPassiveProximity.

            //default to false. Always assume there is no players in the area.
            move 650 %compoundBooleanPart 80 1018 6 -0 0 0
            //move playerProximityActive over the area.
            move 650 %playerProximityActive -600 1020 0 -0 0 0
            //Wait. See in-depth explanation.
            wait 1
            //Signal the boolean
            trigger activate 650 compoundBoolSignal
            wait 1
            move 650 %playerProximityActive -600 1020 -150 -0 0 0
        //compoundActiveProximity
            //pre: playerActiveProximity world part must be triggered. Player-whitelisted.
            //post: Sets the boolean to true regardless of boolean's state.
            //description: Sets boolean to true.

            move 650 %compoundBooleanPart 80 1018 -6 -0 0 0
            //convention
            wait 0.1
            reset
        //compoundBoolSignal
            //pre: prompted by compoundClock.
            //post: spawns two bots over compoundBoolFalse and compoundboolTrue triggers.
            //description: Sends an asynchronous / race condition signal that should be blocked by a world part, resulting in only one output.

            bot spawn 650 PL5_Rifleman 80 1022 6 0
            bot spawn 650 PL5_Rifleman 80 1022 -6 0
            //conv
            wait 0.001
            reset
        //compoundBoolFalse
            //^ This trigger is neccessary. It unloads the area. 
            //pre: compoundBoolTrue must be part-blocked. Must be activated by the bot signals. Bot-whitelisted.
            //post: Cleans the boolean of signals and runs a stop method.
            //description: Begins the shutdown of the area because no players are nearby.
            trigger activate 650 compoundBoolKill
            trigger activate 650 compoundStopMethod2
        //compoundBoolTrue
            //^ This trigger is not neccessary, it just says the players are in the area.
            //pre: CompoundBoolFalse must be part-blocked. Must be activated by the bot signals. Bot-whitelisted.
            //post: Cleans the boolean of signals.
            //description: Does nothing, because players are nearby. 
            trigger activate 650 compoundBoolKill
        //compoundBoolKill
            //^This trigger kills the boolean signals early if not built over the void.
            //Try to build data types over void OW_Ronograd water or other bot killboxes.
            //pre: activated by BoolFalse or BoolTrue, thus ensuring a decision has already been made.
            //post: changes collision status of compoundBooleanPart to kill the bots with an explosion, then resets the triggers to allow for future use by signals.
            //description: refreshes the boolean.
            
            collision 650 %compoundBooleanPart 1
            explosion 18 300 79.000 1017.000 0.000 Flash
            wait 0.01
            collision 650 %compoundBooleanPart 0
            wait 0.01
            trigger reset 650 compoundBoolTrue
            trigger reset 650 compoundBoolFalse
        //compoundStopMethod2
            //pre: false has been triggered.
            //post: 
                //Turns off clock; 
                //Ensures the playerProximityPassive part back in place. This may be redundant if you do not move it in the first place.
                //If there are bots related to the compound, spawn explosions in specific areas to remove these bots;
                //Reset the passive proximity trigger after any and all bots are removed;
                //check world alert statuses;
                //If compound features a raid feature, reset compound raid trigger.
            //description: successful shutdown of area
            
            trigger whitelist 650 compoundClock IsLooping false
            move 650 %playerProximityPassive -600 1020 0 -0 0 0
            explosion 100 300 -600.000 1050.000 0.000 Flash
            trigger reset 650 compoundPassiveProximity
            explosion 50 300 -590 1014 66 Flash
            wait 1
            bot alert 651 false
            trigger reset 650 compoundRaidTrigger

    //Player-Proximity explanations
        //How the world parts "playerProximity" passive and active work
            // playerProximityPassive does not reset in this example, so it should not misfire during a clock interval despite never moving. Regardless, an attempt should be made to move it.
            //- playerProximityPassive is a passive trigger on the main area that loads bots in when player-whitelist triggered. 
            //It starts a clock which moves the playerProximityActive to scan an area encompassing the main area and a buffer zone.
            //if the active proximity trigger zone is player-whitelist triggered, it tells the clock there is a player in the area via the compoundAciveProximity trigger. 
        //compoundClock breakdown
            //default to FALSE
                //playerProximityPassive initiates the first clock signal interval.
                //If the false trigger is not blocked the compoundBoolSignal will activate the false trigger on the first interval.
                //This means the compound/area will pre-maturely unload.
            //compoundClock wait explanation
                //The wait exists to prevent the compoundClock's default boolean set to false.
                // This should never happen due to the order of executables in compoundPassiveProximity.
        //How the player-proximity boolean works
            //By blocking the false trigger you are sending a signal to the true trigger.
            //Conversely if the compoundBooleanPart world object is above the true trigger
            //it allows the false trigger to activate.
            //compoundClock and compoundActiveProximity change the block state blocking one of the two boolean triggers.
            //compoundClock blocks the true trigger by default resulting in a false default.
            //if the active proximity trigger zone does not detect a player this false default will result in FALSE activating the false trigger.
            //if compoundActiveProximity detects a player in its trigger box the part will be moved over the boolean's false trigger repeatedly 
            // until the active proximity zone is removed for that interval. This results in TRUE activating the true trigger.

//#####Glossary#####
    //active: trigger can be activated by whitelisted objects. The trigger is not running. See inactive.
    //alert: see commands reference page "bot alert"
    //area/site: the space designated for things to be done. This can range from spawning bots to moving world objects to having tweening.
    //block-blocked/block state/part state/part-blocked: refers to compoundBoolPart a physical world object that blocks one of two triggers guaranteeing only one trigger is activated.
    //boolean (simplified): Data type with two possible values/states: true; or false. For this project, it is a physical space represented by the space not occupied by the world part.
    //clock signal: https://en.wikipedia.org/wiki/Clock_signal ; but for RGE purposes it's just IsLooping true. Also useful to know the frequency/interval.
    //compound: BRM5's staple system, it is a designated area with enemy bots. This term will be used interchangably with area within this text file, see area.
    //conv/convention: Coding convention / guidelines. A style or practice to prevent common bugs or wtv
    //inactive: trigger cannot be activated by whitelisted objects. The trigger is either in the process of running or is not running. See active.
    //initial/starter/stealth: the bots that first spawn in when a trigger is activated. Terms initial and starter do not imply world is not alerted. See alert,raid
    //pre post description
        //pre: the conditions to activate this trigger
        //post: the result of the trigger
        //description: what the trigger does as a whole
    //raid: an event that happens when the compound is alerted.
    //run/running (triggers): trigger is activating the executable lines.
    //start/load: The designated area is in use by the system. This can range from bots being spawned or world objects being moved or tweened on site.
    //stop/unload: The designated area is not in use by the system. If applicable, any bots are being despawned, or world objects are being moved off site.
    //whitelist: The list of objects that can activate a trigger. This can be players, bots, Ground(vehicles) or helicopters.
    //world part/part/block/part object/world object: a part object in the world tab which can contain props, parts, lighting, interactables, etc.
